<inner> fetchRemote(options)

Fetch the model from the server. If the server's representation of the model differs from its current attributes, they will be overridden, triggering a `"change"` event. Removes localStorage from the collection for the next synch call, then adds it back By the time the callbacks from the fetch are executed, the localStorage will be back in so any new data will be persisted to localStorage. Updating this.localStorage in the complete callback is too late as that happens after all the models are added and localStorage has to be ready. There are at least 3 alternatives: 1) the previous solution was to copy the backbone fetch function in it's entirety, modifying the sync call 2) add something like "remote:true" to the options, and used by backbone localstorage to do the correct sync call 3) add a beforeParse
